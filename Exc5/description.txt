описание элементов схемы:

    Query:
        client(id: ID!): Получить данные о клиенте по его ID, включая вложенные данные о документах и родственниках.
        clients(filter: ClientFilter): Получить список клиентов, используя фильтр для ограничения результатов.

    Client:
        Основные поля для идентификации клиента: id, name, age.
        Вложенные поля:
            documents: Список документов клиента.
            relatives: Список родственников клиента.

    Document:
        Поля для информации о документе клиента: id, type, number, issueDate, expiryDate.

    Relative:
        Поля для информации о родственнике клиента: id, relationType, name, age.

    ClientFilter:
        Позволяет фильтровать список клиентов по полям name (имя) и age (возраст).



Как это помогает оптимизировать взаимодействие между потребителями и сервисом client-info:

    Минимизация числа запросов:
        Вместо выполнения нескольких REST-запросов для получения информации о клиенте, документах и
         родственниках, в GraphQL можно получить все необходимые данные одним запросом.
    Гибкость выборки данных:
        Потребители данных (например, веб-приложение) могут запрашивать только те поля,
        которые им действительно нужны, тем самым сокращая объем передаваемых данных.
    Уменьшение нагрузки:
        С помощью фильтрации и объединения данных в одном запросе, уменьшается нагрузка
        на сервис и сокращается время отклика.
